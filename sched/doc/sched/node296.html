Version 6.04

This release includes everything that was in the 6.03 mini-releases that occurred since 6.02. Those mini-releases were not sent to the broader community and were intended mainly to make available the new OPTMODE=HAS for use by the VIPS survey, which started before we could do a proper SCHED release.

  • If VLA phasing mode scans were NORECORD, SCHED warned about unphased VX mode scans. Change to recognize non-recording phasing scans. 8 Aug 2005.

  • Add station motion adjustments to summary file. 5 Aug 2005 RCW.

  • Fix bug that might cause reading beyond the end of a character string if CORREL is specified as blank. 10 Aug RCW based on information from Cormac.

  • Update Sla/dat.f for the leap second announced for Jan. 1, 2006. 11 Aug. 2005. RCW.

  • Send first message warning of possible unspecified OBSTYPE to the terminal when SCHED  runs. 14 Sept. 2005.

  • Comment a redundant SAVE command in state.f in the Jpl ephemeris routines. This gave gfortran a problem. 16 Sept. 2005

  • Change OPMINANT and OPMINEL to be scan dependent. 25 Oct. 2005

  • Fix the sync time discussion in the PRESTART section. 1 Nov. 2005 RCW

  • Fix dealing with sources when loading a plot parms file when the source list has changed since the parms file was saved. 1 Nov. 2005.

  • Add hour angle based optimization mode OPTMODE=HAS. Initial application is the VLBA Imaging and Polarization Survey (VIPS). 03 Nov 2005 RCW

  • Add the scan comment to the scan information dumped when certain errors occur. 14 Nov 2005 RCW

  • Add note to users of tapeini sections that they normally are not needed and such use betrays the use of ancient templates. 15 Nov 2005 RCW

  • Improve the choice of the maximum separation for the list of source separations. 14 Nov 2005 RCW.

  • Add scan number to summary listings. Add optional scan name based on SCANTAG input variable. 18 Nov 2005 RCW.

  • Protect against having a Vex station with no observations after taking out scans where the source is not up. This befuddles the Vex writer. Warn the user if this happens for any other station. 18 Nov 2005 RCW.

  • Only require shipping address for correlator output if tape is requested. 08 Jan. 2006 RCW.

  • Warn users when it looks like they might have forgotten to reset FREQ or BW when changing setup files. 24 Jan 2006 RCW.

  • Fix bug in overlapped basebands warning when using Doppler calculations. 25 Jan. 2006 RCW.

  • Added CRDLINE to aid development of VLBA on-line software. 26 Jan. 2006 RCW.

  • Improve documentation of reconfigure issues. Jan 27, 2006. RCW.

  • Warn when trying to use 1024 channels, fanout=1, and 16 times oversample on the VLBA correlator. It won't work. Feb. 2006. RCW.

  • Fix selection of the frequency to use when plotting beams. Fix dealing with the save file stations when the station list changes. Other minor plot fixes. Franco Tinarelli and RCW Feb. 2006

  • Add Sec(z) to the XY plot options. Franco Tinarelli Feb. 10, 2006.
