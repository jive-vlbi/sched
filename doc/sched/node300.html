Release of 1 Dec. 1998.

  • Fix bug that affected plotting with planets (SRLSTN not set). 07apr98 RCW.

  • Reactivate OPSKIP and OPELPRIO for pointing observations. 08apr98 RCW.

  • Fix bug in tape footage accounting for automatic tape handling. This put the tape at the wrong end when exactly at one end. 17apr98 RCW.

  • Fix bug in number of tapes listed early in summary file when autotape is used. 23Apr98. RCW.

  • Fix bug that did not allow a setup with two groups, the first with stations=' ' and the second with a specified station. 5may98 RCW.

  • Change the default for PRECDATE to 1979.9, the right value for positions determined on the VLA. Add to the documentation about this issue, including pointing out that MERLIN positions assume an observe date of 1950. 6may98 RCW.

  • Beefed up the warnings in this document about different velocity definitions. 16may98 RCW.

  • For AUTOALLOCATE, fixed the number of passes in the summary to go to count fractional passes. 20may98 RCW.

  • SCHED died with complaints about BBC settings and about duplicate channels when DAR=NONE. Added bypass or those tests with that DAR type. 05jun98 RCW.

  • MAXSTA set at 30 to match test stuff Franco is doing. Keep it there? 16Apr98 RCW. Will keep. 31 Aug 1998. RCW.

  • Update dat.f in Slalib routines for planned leap second at the end of 1998.

  • Add chkcode.f to check that VLA, VLBA, and global schedules have at least 3 digit numbers. 26 Aug. 1998.

  • Add alternate average time. 26 Aug. 1998.

  • Add details to the summary about Sun distance. 26 Aug 1998.

  • Add VDEF and VREF to the source catalog parameters to allow different velocity definitions and different reference frames for velocity. 10 Sept. 1998. RCW.

  • Begin support of VLA-Pie Town link by adding PT LO to the VLA obs file //LO line. 12Sept 1998.

  • Some enhancements to the plotting, especially when there are large numbers of stations. various dates. Franco

  • Fix bug in rotation pattern where the colimation offset adjustments for a rotation were not made for the az part of the pointing raster. 28Oct98 RCW.

  • Update 6.7 and 12.2 GHz methanol frequencies as per email from Chris Phillips. 02Nov98 RCW.

  • Add beam plots. Oct98 Tinarelli.

  • Change an incorrect entry in the okmodes.f DATA statements. Change egiv.set to a valid mode. Add text to the error message when an unapproved mode is used. 17nov98 RCW.

  • Set maximum passes per head position to 8 for defaulted setups. Also deal better with cases that use less than four tracks. 23Nov98 RCW

  • Add check of JIVE correlator options. 23Nov98 Huib.

  • Enhance VEX to deal with more complicated schedules. From Huib: ``The main work (and quite a bit) is that the VEX writer can now deal with more complex schedules. There used to be a type of schedules that used certain stations in different setups in different modes, that could not be dealt with. That is now all resolved, but the price was that I had to maintain a list of stations for every "def", I think the memory use has increased a bit.'' 23Nov98 Huib.

  • Default barrel=roll_8 (or at least not roll_16) when the user wants 1024 channels on the VLBA correlator. 23Nov98. RCW.

  • Increase by 1 digit the precision to which RA and Dec are printed in the summary, crd, and sch files. 23Nov98 RCW.

  • Fix some bugs in the call to CHKGDAR and in CHKDAR itself. 30Nov98 RCW.
