Release of 1 July 1999.

  • CRDVLA temporarily stored FLUKEA and FLUKEB in single precision which led to round off problems in some special cases. Fixed 04Jan98 RCW.

  • Fix translation of VLA bandwidth code into an actual bandwidth. Was BW=50/(n+1). Should be BW=50/2**n. 15jan99 RCW.

  • Franco fixed a bug that caused improper reinitialization of LST plot limits when a RESTART is done. 26jan99.

  • Got rid of special Plot_Tinarelli directory. Franco's stuff is now in the Plot directory. The old, pre GUI, plot stuff was tarred and put in the $SCHED/OLD directory. 26jan99 RCW.

  • Fixed bug in SETFCAT, the routine that chooses the frequency catalog entry to use. In choosing between two entries with good frequencies, it would choose the lower priority (higher numerical value of PRIORITY) one if the observing frequency was better centered. 28jan99 RCW.

  • Changed gap required for a readback to 1:45. For the on-line system, it is really 1 minute plus 1 second per track and it can't be too close to the start of a pass (SCHEDdoes not test that). This change was triggered because the old 2:00 test, combined with LST scheduling, was causing lots of readback warnings that were not needed. 8feb99 RCW.

  • Blocked the warning about scans crossing 0 hr UT. This is no longer a concern, as far as I can tell. The Socorro correlator will put a job boundary there, but this should be benign. 16feb99 RCW.

  • Allow the use of UT dates (YEAR, MONTH, DAY) for an LST schedule. This might make dynamic scheduling easier. Cleaning up code relating to setting scan times (before optimization). 17feb99 RCW.

  • Count the number of times the tape stops. If it is more than 10 times per hour, warn the user about excessive stops and point out new parameter MINPAUSE. 17feb99 RCW.

  • Add parameters MINPAUSE and PRESTART. PRESTART causes the start of tape motion to be moved forward to allow for synchronization at playback time. MINPAUSE causes the use of continuous tape motion if the tape stop time would between 2 scans would be less than MINPAUSE times the speed up factor. Related to this, there are significant updates to the manual, especially the sections on scan times. 24feb99 RCW.

  • Add warning for VLBA correlator projects going to the VLBA correlator if AUTOTAPE=2 is not being used. 22feb99 RCW.

  • Add warning when DOPPLER set freuqencies overlap more than 20
  • Update OK modes. Now almost all are allowed. 26Feb99 RCW

  • Remove UPTIME from the SCHED  distribution. 01Mar99 RCW.

  • Add concept of frequency sets. This is to aid bookkeeping when the deviations from the setup file frequencies are altered with FREQ, BW, or DOPPLER. Now in the summary file, all frequency sets are listed along with the setup file summary. 2mar99 RCW.

  • Altered VLBI at VLA example in manual to show full specification of VLA parameters. 3Mar99 RCW.

  • Adding BAND for setups. This should allow elimination of most standard setup files. 4Mar99 RCW

  • Added pcal sets to organize the pulse cal detector specifications. This is mainly to simplify bookkeeping, especially for Vex. 13Mar99 RCW.

  • Added a .flag file output that can be used to flag predicted slew times with UVFLG. 13Mar99 RCW.

  • Remove stations from a scan when they are both using autotape=2 and the source is not up. This is to help reduce tape consumption, especially for MK. It is only done for automatic tape allocation because of the risk of messing up tape management in other cases. 18mar99. RCW.

  • Add VLBA4 to the DAR options. Started 24mar99.

  • Add VLAPEAK parameter. Could not keep united peakup options and allow the various VLA reference pointing options. 30mar99 RCW.

  • Don't make a crd.y file when scheduling pure VLA observations. 31mar99 RCW.

  • When dwell time scheduling was in use, SCHED  would not make scans that cross midnight UT. Don't bother any more -- allow scans to cross midnight. 31mar99 RCW.

  • Stop tape at end of autoallocation projects. 12May1999 RCW.

  • Updates of the DUR and DWELL sections of the manual. 17may99 RCW.

  • Add a cross check for reasonableness of the tape length and density specification for VLBA tapes. Huib may add same for Mark IV. 17may99 RCW.

  • Prevent missing samplerate in VLBA schedules when first scan is pointing. 7jun99 RCW.

  • While SCHED  is running, write out the full time range to help when fine tuning to fill a time allocation. This from Ketan. 13Jan99.

  • VEX section upgraded to 1.5.60 21Jun99, Huib
    S2 tape changes corrected
    Checks use narrow filters in MkIV DAR
    Upgraded to much more robust freq change scheme
    BW changes now allowed in VEX
    Freq/Dop changes in modes with different BBC assignment allowed
    Reduced memory use VEX section
    Check tape starts synchronous for VEX
    Axis offsets, negative Vlsr now reported
    Check against high density recording on thick tapes

  • Prevent different tape start times for a scan when a VEX output is requested. RCW 20jun99

  • Improve the count of scans (remove bug). RCW 20jun99

  • Put experiment code in upper case on VLA observe deck. RCW 20jun99.

  • Don't die if S2 or other format requested for processing in Socorro - allows translators. 29June1999 RCW.
